2. Kiến Trúc Hệ Thống

Giống cấu trúc của file presentation tham chiếu, hệ thống được trình bày theo dạng modular monolith với các domain tách rõ trách nhiệm. Khác biệt chính là backend ở đây dùng ASP.NET Core Web API, truy cập dữ liệu qua EF Core + Npgsql, và dùng SignalR cho các cập nhật realtime.

3. Luồng Khởi Động (Backend + Frontend)

Phần này được viết theo tinh thần của file presentation nguồn: mô tả rõ hệ thống boot như thế nào, nhưng toàn bộ bước kỹ thuật đã đổi sang stack .NET 8 và PostgreSQL.

Backend Startup Sequence

Tầng 1 — Configuration Guard

Kiểm tra ConnectionStrings__Default, JWT secrets, cookie settings, CORS origins và cấu hình SignalR trước khi mở app.

Tầng 2 — PostgreSQL Connectivity

Khởi tạo DbContext, kiểm tra kết nối PostgreSQL qua Npgsql, bật retry policy nếu cần.

Tầng 3 — EF Core Migration / Indexing

Áp dụng migration, đảm bảo unique index cho users.email, qr_codes.code, và các foreign key giữa pois, menus, tours.

Tầng 4 — Seed Roles & Admin

Tạo sẵn role admin, poi_owner, tourist; seed tài khoản quản trị nếu môi trường yêu cầu.

Tầng 5 — Endpoint + Hub Ready

Map controllers/API groups, mount /hubs/notifications, bật auth middleware, static files và map style endpoints.

Frontend Startup Flow

App Bootstrap → GET /auth/me → Login Gate → Load /map → Connect SignalR

🔐 Session Restore

Frontend đọc session qua /auth/me; nếu chưa xác thực thì chuyển thẳng tới /login.

📍 Initial Data Load

Sau khi đăng nhập, app nạp /pois, tour publish và menu tóm tắt để render map + landing state.

📡 Realtime Attach

signalRClient.start() được gọi ngay sau khi có session để lắng nghe menu/tour/QR events.

8. Owner / Admin / Realtime Operations

Đây là phần lấy cảm hứng trực tiếp từ section vận hành của file presentation mẫu, nhưng đã đổi sang mô hình tạm thời hiện tại: owner quản lý POI/menu, admin quản lý toàn cục, và SignalR đẩy thay đổi tới client đang online.

🏪 Owner Portal

Owner chỉ thao tác trên POI được gán: sửa thông tin quán, menu food, xem QR gắn với quán hoặc tour liên quan.

🛡️ Admin Dashboard

Admin quản lý users, roles, POIs, menus, tours, QR codes và audit logs; đồng thời là nguồn phát event realtime chính.

📡 SignalR Broadcast

Mỗi thay đổi đã commit vào PostgreSQL sẽ phát event như menuUpdated, tourPublished, qrScanned để frontend refresh đúng chỗ.

Operational Flow

Owner/Admin Edit → Save PostgreSQL → Write Audit Log → Broadcast SignalR → Client Refresh
Góc nhìn tạm thời: tài liệu đang ưu tiên mô hình quản trị và discovery flow qua POI + menu + tour + QR, thay cho luồng audio/localization của file presentation gốc.

9. Core Design Patterns

🔐 Login-Gated Visitor Flow

App start → check /api/auth/me → nếu chưa đăng nhập thì redirect /login → login success mới vào /map, /tours, /qr/* → role-based route guard cho owner/admin

📡 SignalR Realtime Sync

Owner/Admin update POI, menu, tour hoặc QR → API save vào PostgreSQL → Hub broadcast event → client đang online nhận menuUpdated / tourPublished / qrScanned → refresh đúng màn hình đang mở

🔳 QR Deep-Link Routing

Scan QR → resolve code trong PostgreSQL → xác định target_type: poi | menu | tour → kiểm tra auth → điều hướng tới màn hình tương ứng

🧭 POI + Menu + Tour Ownership

Admin gán POI cho POI_owner → owner chỉ sửa được menu/POI của mình → tour có thể gom nhiều POI theo thứ tự stop → QR code có thể trỏ vào POI, menu hoặc tour → audit log lưu mọi thay đổi quan trọng

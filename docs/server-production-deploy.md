# Server Production Deploy

This repo now has a helper script for preparing a deployable server artifact:

- [scripts/deploy_server.ps1](D:/VinhKhanhFoodStreet/scripts/deploy_server.ps1)
- [scripts/install_and_run_server.ps1](D:/VinhKhanhFoodStreet/scripts/install_and_run_server.ps1)
- [scripts/install_iis_prerequisites.ps1](D:/VinhKhanhFoodStreet/scripts/install_iis_prerequisites.ps1)

## 1. Publish the server artifact

Production publish:

```powershell
.\scripts\deploy_server.ps1 -Environment Production
```

Staging publish:

```powershell
.\scripts\deploy_server.ps1 -Environment Staging
```

Output:

- folder: `artifacts\server-publish\<environment>`
- zip: `artifacts\server-publish\narrationapp-server-<environment>.zip`

## 2. Copy artifact to the target server

Copy either:

- the full published folder
- or the zip generated by the script

If the target is a Windows server, you can also use the installer helper directly after copying the repo or the publish artifact:

```powershell
.\scripts\install_and_run_server.ps1 `
  -Environment Production `
  -SourcePath .\artifacts\server-publish\narrationapp-server-production.zip `
  -InstallRoot C:\apps\narrationapp-server `
  -ConnectionString "Host=...;Port=5432;Database=...;Username=...;Password=..." `
  -PublicQrBaseUrl "https://narration.app/" `
  -StartMode Background
```

This script:

- extracts or copies the artifact
- backs up the previous `current` deployment
- installs the new build under `current`
- creates `run-server.ps1`
- optionally starts the server in foreground or background

## 3. Set required runtime variables on the server

Minimum:

```text
ASPNETCORE_ENVIRONMENT=Production
ASPNETCORE_URLS=http://0.0.0.0:5000
ConnectionStrings__PostgreSql=Host=...;Port=5432;Database=...;Username=...;Password=...
```

If needed, you can also override:

```text
PublicQr__BaseUrl=https://narration.app/
```

## 4. Start the server

From the published folder:

```powershell
.\NarrationApp.Server.exe
```

Or, if you used the Windows installer helper:

```powershell
powershell -ExecutionPolicy Bypass -File C:\apps\narrationapp-server\run-server.ps1 -Mode Background
```

## 5. Put the app behind the public HTTPS domain

Reverse proxy the app so these URLs are live:

- `https://narration.app/qr/<code>`
- `https://narration.app/.well-known/assetlinks.json`

Repo samples:

- [docs/reverse-proxy-setup.md](D:/VinhKhanhFoodStreet/docs/reverse-proxy-setup.md)
- [deploy/reverse-proxy/nginx/narrationapp.conf](D:/VinhKhanhFoodStreet/deploy/reverse-proxy/nginx/narrationapp.conf)
- [deploy/reverse-proxy/iis/web.config](D:/VinhKhanhFoodStreet/deploy/reverse-proxy/iis/web.config)
- [scripts/configure_iis_reverse_proxy.ps1](D:/VinhKhanhFoodStreet/scripts/configure_iis_reverse_proxy.ps1)

## 6. Verify QR + app links

Check:

1. `https://narration.app/.well-known/assetlinks.json` returns JSON.
2. `https://narration.app/qr/<sample-code>` opens the public POI page.
3. Android device with the signed app installed opens the app from the QR HTTPS link.
4. Device without the app stays on the public web page.
